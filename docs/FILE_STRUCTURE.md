## Current Source Areas
- `server/server.js` starts Express, WebSocket support, and route registration.
- `server/db.js` initializes SQLite and handles migrations.
- `server/routes/admin.js` manages host sessions, join flow, prompts, and game control.
- `server/routes/games.js` exposes public read endpoints and WebSocket broadcasts.
- `server/routes/prompts.js` provides prompt CRUD.
- `server/lib/game-helpers.js` contains pure helper logic covered by tests.
- `client/src/App.jsx` selects the host, join, and board views.
- `client/src/pages/HostDashboard.jsx` manages host game creation and prompt control.
- `client/src/pages/JoinGame.jsx` handles QR scanning, name entry, team selection, and captain selection.
- `client/src/pages/GameBoard.jsx` renders the live board, roll action, prompt card, and host controls.

## Test and Utility Files
- `server/tests/game-helpers.test.js` is the current automated test suite.
- `server/test-board.js` and `test-board.js` are legacy manual smoke-test scripts.
- `npm test` runs the automated helper suite.

## Notes
- `node_modules` directories are generated by npm and should not be committed.
- `server/momentum.db` is generated locally and should be backed up or ignored in deployment workflows.
